3) Subnet-lijst (van eerst naar laatst)

Met netwerk 192.168.0.0/24 en gekozen prefix /26:

Subnet nummer	Subnetadres	Prefix	Subnetmask (dotted)	Geldige host range		Broadcast
1 (LAN-A)	192.168.0.0	/26	255.255.255.192		192.168.0.1 – 192.168.0.62	192.168.0.63
2 (LAN-B)	192.168.0.64	/26	255.255.255.192		192.168.0.65 – 192.168.0.126	192.168.0.127
3 (reserve)	192.168.0.128	/26	255.255.255.192		192.168.0.129 – 192.168.0.190	192.168.0.191
4 (reserve)	192.168.0.192	/26	255.255.255.192		192.168.0.193 – 192.168.0.254	192.168.0.255


4) Vul de Addressing Table in (belangrijkste velden)

Ik vul alleen de lege velden in uit jouw oorspronkelijke tabel (ISP-zijde laat ik zoals gegeven). Ik gebruik de toewijzingen volgens opdracht (eerste host = router G0/x, tweede host = switch VLAN1, laatste host = PC).

LAN-A (subnet 192.168.0.0/26)
	Subnetmask: 255.255.255.192
	CustomerRouter G0/0: 192.168.0.1 /255.255.255.192 (Default gateway for hosts on LAN-A)
	LAN-A Switch VLAN1: 192.168.0.2 /255.255.255.192 (Default gateway: 192.168.0.1)
	PC-A NIC: 192.168.0.62 /255.255.255.192 (Default gateway: 192.168.0.1)

LAN-B (subnet 192.168.0.64/26)
	Subnetmask: 255.255.255.192
	CustomerRouter G0/1: 192.168.0.65 /255.255.255.192 (Default gateway for hosts on LAN-B)
	LAN-B Switch VLAN1: 192.168.0.66 /255.255.255.192 (Default gateway: 192.168.0.65)
	PC-B NIC: 192.168.0.126 /255.255.255.192 (Default gateway: 192.168.0.65)

Overige (al ingevuld in jouw tabel)
	CustomerRouter S0/1/0 = 209.165.201.2 /30 (zoals gegeven)
	ISP Router & apparaten blijven zoals in jouw tabel (209.165.200.x etc.)
Opmerking: ik heb pc-adressen gekozen volgens jouw instructie (laatste host in subnet voor pc). Je kunt natuurlijk andere vrije host-adressen binnen het subnet kiezen, maar volg de regels van de opdracht.
